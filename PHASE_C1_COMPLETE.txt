WHAT WAS IMPLEMENTED

Module: ufa/optimizer/ticket_governor.py (297 lines)
- Class: TicketGovernor
- Purpose: Enforce exposure limits within individual parlay tickets (per-ticket state)
- Architecture: Stateless evaluation (input ticket → verdict + adjustments)

Core Methods:
  1. evaluate_ticket(picks: List[Dict]) → TicketAdjustment
     - Evaluates single parlay for exposure violations
     - Returns: (verdict, message, penalty_mult, warnings, notes)
  
  2. batch_evaluate(ticket_list) → List[TicketAdjustment]
     - Evaluates multiple tickets (no cross-ticket state)
  
  3. format_summary(adjustments) → str
     - Formats summary for cheatsheet or logging

Verdict Types:
  - APPROVED: Clean ticket, no penalties
  - APPROVED_WITH_PENALTY: Acceptable but penalized
  - DOWNGRADED: Exposure violation, tier reduction
  - BLOCKED: Hard gate violation, 0 EV

================================================================================
EXPOSURE RULES (HARD GATES)
================================================================================

HARD BLOCKS (Verdict = BLOCKED, Penalty = 0%):
  1. 3+ alt-stats in same ticket
     - Example: "Pass attempts" + "Rush attempts" + "Targets" → BLOCKED
     - Reason: Overconcentration of derived stats
  
  2. 2+ early-sequence stats in same ticket
     - Example: "Points (First 3 Min)" + "Assists (First 3 Min)" → BLOCKED
     - Reason: High correlation in early-game windows
  
  3. 2+ event-binary stats in same ticket
     - Example: "Longest Play > 40 yards" + "Dunk Hit" → BLOCKED
     - Reason: Rare events are independent; can't correlate
  
  4. 2+ same-player in same ticket
     - Example: "Lamar Pass Attempts" + "Lamar Pass Yards" → BLOCKED
     - Reason: Cannot double-dip on single player

SOFT GATES (Verdict = APPROVED_WITH_PENALTY or DOWNGRADED):
  1. 2 alt-stats in same ticket
     - Penalty: 95% EV multiplier (5% reduction)
     - Example: "Pass attempts" + "Core points" → Approved but 5% EV cut
  
  2. Early-sequence + Event-binary mixed
     - Penalty: 90% EV multiplier (10% reduction)
     - Reason: Independent stat classes reduce edge
  
  3. 2 same-team in same ticket
     - Penalty: 85% EV multiplier (15% reduction)
     - Reason: Same-team correlation reduces independence
  
  4. 3 same-team in same ticket
     - Verdict: DOWNGRADED
     - Penalty: 70% EV multiplier (30% reduction)
     - Reason: Heavy correlation; tier downgrade mandatory

================================================================================
KEY DESIGN DECISIONS
================================================================================

1. STATELESS EVALUATION
   - Each ticket is evaluated independently
   - No memory of previous tickets or cross-ticket state
   - Safe to run on any slate without side effects

2. MULTIPLICATIVE PENALTIES
   - Penalties multiply (not add)
   - Prevents "penalty cliff" at boundaries
   - Example: 0.95 × 0.85 = 0.808 (not 0.95 + 0.85)

3. SAME-TEAM HEURISTIC
   - Uses team field from pick dict
   - 2 same-team = soft penalty (15%)
   - 3 same-team = hard downgrade (30%)
   - Same-game correlation captured by existing tagger

4. VERDICTS DON'T CHANGE TIER
   - DOWNGRADED is descriptive (for messaging)
   - Actual tier changes handled by calibrator
   - Ticket governor is pure penalty mechanism

5. BLOCKING IS IRREVERSIBLE
   - If hard gate violated (2+ early, 2+ event, etc.) → 0% EV
   - No exceptions (phase C-2 won't add slate-level recovery)
   - Forces users to restructure ticket
