WHAT C-1 DOES (QUICK REFERENCE)
================================================================================

BLOCKS these tickets (0% EV):
  • 3+ alt-stats in same parlay (e.g., pass_att + rush_att + targets)
  • 2+ early-sequence stats (e.g., first-3-min points + first-3-min assists)
  • 2+ event-binary stats (e.g., dunk + steal)
  • 2+ same-player picks (e.g., Lamar pass_yds + Lamar pass_att)

PENALIZES these combinations:
  • 2 alt-stats: 5% EV cut (0.95x multiplier)
  • Early + Event mixed: 10% EV cut (0.90x multiplier)
  • 2 same-team: 15% EV cut (0.85x multiplier)
  • 3 same-team: 30% EV cut (0.70x multiplier) + tier downgrade

Core picks (stat_class="core") are **never** blocked or penalized.

QUESTIONS / CONCERNS
================================================================================

Q: What if a good ticket gets blocked?
A: Phase C-2 may add override logic or allow user exceptions. For now, 
   blocking is strict (safety-first approach).

Q: Can I disable C-1?
A: Yes, comment out the ticket_governor.evaluate_ticket() call in 
   _generate_parlay_section(). Or set penalty_mult = 1.0 always.

Q: What if alt-stats aren't working yet (not ingesting alt-stat picks)?
A: C-1 won't block anything (all picks are "core" now). Once alt-stats 
   are ingested, C-1 will activate automatically.

Q: How do I know if C-1 is working?
A: Check cheatsheet output. If you see "⚠️ Approved (with penalty)" 
   or "❌ BLOCKED", C-1 is active. If all tickets say "✅ APPROVED", 
   either no alt-stats ingested or all tickets are clean.
